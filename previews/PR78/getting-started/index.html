Getting started

Setup

OMFrontend depends on several sibling packages from the OM.jl project. The simplest way to develop against it is to clone the umbrella repository, which carries each sibling as a git submodule:

git clone --recursive https://github.com/JKRT/OM.jl
cd OM.jl/OMFrontend.jl
julia --project -e 'import Pkg; Pkg.instantiate()'

When using the registered package directly, Pkg.add("OMFrontend") resolves the same dependency graph through OpenModelicaRegistry.

Parsing a Modelica file

using OMFrontend
program = OMFrontend.parseFile("MyModel.mo")

parseFile invokes the OMParser bridge and returns an Absyn.Program. The Absyn tree is the closest representation to the original textual model.

Lowering to SCode

scodeProgram = OMFrontend.translateToSCode(program)

SCode is a lowered, modification-free representation. Most semantic checks operate on SCode rather than Absyn.

Producing a DAE

(dae, cache) = OMFrontend.instantiateSCodeToDAE("MyModel", scodeProgram)

The instantiation pass elaborates the chosen model, resolves modifications, expands inheritance, and emits a DAE.DAElist together with the type cache used during lookup.

Producing flat Modelica

For tooling that wants a textual, fully-flat view of the model:

(flat, cache) = OMFrontend.instantiateSCodeToFM("MyModel", scodeProgram)
