OMFrontend.jl

OMFrontend is the Julia port of the OpenModelica frontend. It parses Modelica source, lowers it through SCode, and produces the DAE / flat-Modelica representation consumed by OMBackend.jl and the wider OM.jl toolchain.

What this package does

  • Parses Modelica via OMParser.jl and produces an Absyn tree.
  • Translates the abstract syntax tree to SCode.
  • Instantiates SCode into the DAE.jl representation suitable for backend lowering.
  • Optionally produces flat Modelica for inspection or downstream use.

Installation

OMFrontend lives in the OpenModelicaRegistry. Add the registry once, then install the package:

using Pkg
Pkg.Registry.add("General")
Pkg.Registry.add(Pkg.RegistrySpec(url = "https://github.com/JKRT/OpenModelicaRegistry.git"))
Pkg.add("OMFrontend")

Quick example

using OMFrontend

p = OMFrontend.parseFile("MyModel.mo")
scode = OMFrontend.translateToSCode(p)
(dae, cache) = OMFrontend.instantiateSCodeToDAE("MyModel", scode)

See Getting started for a longer walk-through and the API reference for the full list of public entry points.

src/OMFrontend.jlTop-level module: re-exports, precompilation.

src/FrontendUtil/Shared utilities used by the frontend passes.
src/Util/Hash maps, AVL trees, flag handling, etc.

src/SCodeUtil.jlSCode helpers and traversals.
src/main.jlConvenience entry points (parseFile, ...).
